# 快速开始指南

## 1. 环境准备

### 1.1 安装 Flutter

访问 [Flutter 官网](https://flutter.cn/) 下载并安装 Flutter SDK。

**Windows 用户**:
```bash
# 下载 Flutter SDK
# 解压到目标目录，如 C:\flutter

# 添加到环境变量 PATH
C:\flutter\bin

# 验证安装
flutter doctor
```

**macOS 用户**:
```bash
# 使用 Homebrew 安装
brew install flutter

# 验证安装
flutter doctor
```

### 1.2 安装开发工具

推荐使用以下 IDE：

**Android Studio**:
- 下载地址: https://developer.android.google.cn/studio
- 安装 Flutter 和 Dart 插件

**VS Code**:
- 下载地址: https://code.visualstudio.com/
- 安装 Flutter 和 Dart 扩展

### 1.3 配置 Android 开发环境

1. 安装 Android Studio
2. 安装 Android SDK
3. 配置 Android 模拟器或连接真机

```bash
# 检查 Android 环境
flutter doctor --android-licenses
```

### 1.4 配置 Windows 开发环境（可选）

如果需要开发 Windows 版本，需要安装 Visual Studio 2022：

1. 下载 Visual Studio 2022
2. 安装时选择"使用 C++ 的桌面开发"工作负载
3. 确保包含以下组件：
   - MSVC v142 或更高版本
   - Windows 10 SDK

```bash
# 检查 Windows 环境
flutter doctor
```

```bash

```


## 4. 项目结构说明

### 4.1 核心目录

```
opensna/
├── lib/
│   ├── core/              # 核心功能
│   ├── features/          # 功能模块
│   ├── shared/            # 共享资源
│   └── main.dart          # 入口文件
├── assets/                # 资源文件
├── test/                  # 测试文件
└── pubspec.yaml           # 依赖配置
```

### 4.2 功能模块结构

每个功能模块采用 Clean Architecture：

```
features/auth/
├── data/                  # 数据层
│   ├── models/            # 数据模型
│   ├── datasources/       # 数据源
│   └── repositories/      # 仓库实现
├── domain/                # 业务逻辑层
│   ├── entities/          # 实体
│   ├── repositories/      # 仓库接口
│   └── usecases/          # 用例
└── presentation/          # 展示层
    ├── pages/             # 页面
    ├── widgets/           # 组件
    └── providers/         # 状态管理
```

## 5. 开发流程

### 5.1 创建新功能

1. 在 `lib/features/` 下创建功能目录
2. 按照 Clean Architecture 组织代码
3. 在 `AppRoutes` 中注册路由
4. 在 Dashboard 中添加入口

### 5.2 添加依赖

编辑 `pubspec.yaml`:

```yaml
dependencies:
  new_package: ^1.0.0
```

然后运行:

```bash
flutter pub get
```

### 5.3 添加资源文件

1. 将文件放到 `assets/` 目录
2. 在 `pubspec.yaml` 中声明:

```yaml
flutter:
  assets:
    - assets/images/logo.png
```

3. 在代码中使用:

```dart
Image.asset('assets/images/logo.png')
```

### 5.4 状态管理

**使用 Provider**:

```dart
// 1. 创建 Provider
class MyProvider extends ChangeNotifier {
  int _count = 0;
  int get count => _count;
  
  void increment() {
    _count++;
    notifyListeners();
  }
}

// 2. 注册 Provider
MultiProvider(
  providers: [
    ChangeNotifierProvider(create: (_) => MyProvider()),
  ],
  child: MyApp(),
)

// 3. 使用 Provider
final provider = context.watch<MyProvider>();
Text('Count: ${provider.count}')
```

**使用 Riverpod**:

```dart
// 1. 创建 Provider
final counterProvider = StateProvider<int>((ref) => 0);

// 2. 使用 Provider
class MyWidget extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final count = ref.watch(counterProvider);
    return Text('Count: $count');
  }
}
```

### 5.5 网络请求

```dart
// 使用 ApiClient
final response = await ApiClient().get('/api/user/info');

// 处理响应
if (response.statusCode == 200) {
  final data = response.data;
  // 处理数据
}
```

### 5.6 路由导航

```dart
// 跳转到命名路由
Navigator.pushNamed(context, AppRoutes.login);

// 跳转并传递参数
Navigator.pushNamed(
  context,
  AppRoutes.detail,
  arguments: {'id': 123},
);

// 返回上一页
Navigator.pop(context);

// 替换当前页面
Navigator.pushReplacementNamed(context, AppRoutes.home);
```

## 7. 构建发布

### 7.1 Android 构建

```bash
# 构建 APK
flutter build apk --release

# 构建 App Bundle（推荐）
flutter build appbundle --release

# 输出位置
# APK: build/app/outputs/flutter-apk/app-release.apk
# AAB: build/app/outputs/bundle/release/app-release.aab
```

### 7.2 Windows 构建

```bash
# 构建 Windows
flutter build windows --release

# 输出位置
# build/windows/runner/Release/
```

### 7.3 签名配置

**Android**:

编辑 `android/app/build.gradle`:

```gradle
android {
    signingConfigs {
        release {
            storeFile file("your-keystore.jks")
            storePassword "your-password"
            keyAlias "your-alias"
            keyPassword "your-password"
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

**Windows**:

Windows 应用不需要特殊签名配置，但可以使用代码签名证书。

## 8. 常见问题

### 8.1 依赖冲突

```bash
# 清理缓存
flutter clean
flutter pub get

# 升级依赖
flutter pub upgrade
```

### 8.2 模拟器问题

```bash
# 列出所有模拟器
flutter emulators

# 启动模拟器
flutter emulators --launch <emulator_id>

# 冷启动模拟器
flutter emulators --launch <emulator_id> --cold-boot
```

### 8.3 真机调试

**Android**:
1. 开启开发者选项
2. 开启 USB 调试
3. 连接电脑并授权

**Windows**:
直接在本机运行即可。

### 8.4 网络请求失败

检查网络权限配置：

**Android** (`android/app/src/main/AndroidManifest.xml`):
```xml
<uses-permission android:name="android.permission.INTERNET" />
```

**Windows**:
Windows 应用默认有网络权限，无需额外配置。
